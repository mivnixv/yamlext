yamlext

A CLI tool that extends YAML with !include and !merge custom tags.

Quick start View on GitHub
curl -fsSL https://raw.githubusercontent.com/mivnixv/yamlext/main/install.sh | sh

Features

📂

!include

Inline any YAML file — or extract a specific nested field or sequence index from it.

🔀

!merge

Deep-merge multiple mapping files or concatenate sequences, left-to-right.

⚡

Fast & portable

Single binary. Pre-built for Linux, macOS, and Windows — no runtime required.

🔁

Recursive

Included files can themselves use !include and !merge — chains resolve to any depth. Circular includes are detected and reported as errors.

Quick start

curl -fsSL https://raw.githubusercontent.com/mivnixv/yamlext/main/install.sh | sh
yamlext input.yaml > output.yaml
yamlext input.yaml --base-dir /path/to/base > output.yaml

--base-dir overrides the directory used to resolve relative paths in tags (defaults to the input file's directory).

Custom Tags

!include — entire file

config: !include data/base.yaml

!include — specific nested field

city: !include [data/tags.yaml, "address/city"]

!include — sequence index

# Pull a field from the second item in a sequence
second_user: !include [data/users.yaml, "1/name"]

!merge — mappings (deep, left-to-right)

merged_db: !merge [data/base.yaml, data/overrides.yaml, data/extra.yaml]

!merge — sequences (concatenated)

all_items: !merge [data/items.yaml, data/more_items.yaml]

!merge — root-level

# The entire document becomes the merged result
!merge [data/base.yaml, data/overrides.yaml]

!include notes

  • Paths are relative to the file containing the tag (so included files can have their own relative includes).
  • Field paths use / as separator: "address/city", "users/1/name".
  • Numeric path segments are treated as sequence indices: "replicas/0/host".

!merge notes

  • Mappings: later files override earlier ones, recursively.
  • Sequences: items are appended in order.
  • Merging a mapping with a sequence is an error.
